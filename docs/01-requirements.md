# 要件定義ドラフト（v0）

## 目的
- 日々のタスクを素早く整理し、進捗を把握できる個人向けタスク管理。

## 対象ユーザー
- 単一ユーザー（認証なし、ローカル利用のみ）。

## 主要ユースケース
- タスクを作成する
- タスク一覧を確認する
- タスクを完了/未完了に切り替える
- 不要なタスクを削除する
- 期限や優先度で絞り込む（任意）

## ユーザーストーリー
- 仕事前に今日やることを素早く登録したい。
- 重要なタスクだけを優先的に見たい。
- 期限切れのタスクを見落とさずに確認したい。
- 完了済みタスクを非表示にして集中したい。

## 受け入れ条件（MVP）
- タスク作成
  - タイトル必須。空欄の場合は作成できない。
  - 作成時に `createdAt` が自動で設定される。
  - 作成後は一覧の先頭（作成日時の新しい順）に表示される。
- タスク一覧
  - 完了/未完了が明確に区別できる。
  - 期限がある場合は表示される。
  - 優先度がある場合は表示される。
- 完了切り替え
  - トグル操作で完了/未完了が即時反映される。
  - 完了状態はリロード後も保持される。
- 編集
  - タイトルは空欄にできない。
  - 編集内容は保存後に一覧へ即時反映される。
- 削除
  - 1件単位で削除できる。
  - 削除後は一覧から即時消える。
- フィルタ/ソート
  - 完了/未完了フィルタを切り替えられる。
  - 期限順/優先度順のソートができる。
  - フィルタ/ソートの選択状態は保持される（ページ内）。
- 永続化
  - リロード後もタスクが保持される（LocalStorage）。

## 機能要件（MVP）
- タスクCRUD
  - 追加: タイトル、説明（任意）、期限（任意）、優先度（任意）
  - 一覧: 完了状態、期限、優先度を表示
  - 更新: 完了トグル、内容編集
  - 削除: 1件削除
- フィルタ/ソート
  - 完了/未完了でフィルタ
  - 期限/優先度でソート
- ローカル保存
  - LocalStorage（初期） / IndexedDB（将来）

## 仕様詳細
- フィルタ
  - 既定は「すべて」。
  - 「未完了」「完了」を切り替え可能。
- ソート
  - 既定は作成日時の新しい順。
  - 期限順は「期限ありを上、期限なしを下」。
  - 優先度順は「high > medium > low > 未設定」。
- 期限の扱い
  - 期限は日付のみ（時刻なし）で入力する。
  - 表示は `YYYY-MM-DD` を基本とする。
- 編集UI
  - インライン編集（カード内で編集できる）。

## 非機能要件
- レスポンシブ（モバイル/デスクトップ）
- 初期表示は概ね1秒以内を目標
- 余白・カード・タイポグラフィを重視したモダンUI

## 画面構成
- 1画面完結
  - ヘッダー: アプリ名、フィルタ/ソート
  - メイン: 入力フォーム + 一覧
  - 一覧はカードレイアウト

## データモデル（案）
- Task
  - id: string
  - title: string
  - description?: string
  - dueDate?: string（ISO）
  - priority?: "low" | "medium" | "high"
  - completed: boolean
  - createdAt: string（ISO）

## バリデーション/制約
- タイトルは1〜80文字程度を想定。
- 説明は任意、上限は数百文字程度を想定。
- 期限は過去日でも登録可能（警告表示は任意）。

## 例外・エラーハンドリング
- LocalStorage が利用できない場合はメモリ上に保持し、警告を表示する。
- 破損した保存データは初期化して継続できるようにする。

## 対象外（v0）
- 認証/ユーザー管理
- 共有/コラボ
- 通知
